Планировщик задач (Task Scheduler)

Этот проект представляет собой веб-сервис для управления списком задач с поддержкой повторений. Он позволяет добавлять, просматривать, редактировать, удалять и выполнять задачи. Для каждой задачи можно указать дату, заголовок, комментарий и правило повторения (например, ежедневно, еженедельно и т.д.).

Сервер написан на чистом Go (без фреймворков), данные хранятся в SQLite. Проект реализует REST-like API и может использоваться как бэкенд для веб- или мобильного приложения.

Реализованные функции

1. Создание веб-сервера — да
2. Проектирование и создание БД — да
3. Написание функции для вычисления следующей даты у задачи — да
4. Реализация обработчика для добавления задачи — да
5. Получение списка ближайших задач — да
6. Реализация обработчика для редактирования задачи — да
7. Реализация обработчиков при выполнении задачи и её удалении — да
8. Подключение аутентификации — нет (опционально)

Дополнительные возможности

- Автоматическое вычисление следующей даты:
При выполнении задачи с полем repeat вызывается функция, которая вычисляет следующую дату выполнения по правилу.
Поддерживаются правила: daily, weekly, monthly, yearly, а также X (где X — число дней).
Флаг FullNextDate = true — означает, что вычисление даты учитывает полную логику (включая сложные случаи, например, переход через конец месяца).

- Индексы в БД:
Индекс по полю date для ускорения выборки ближайших задач.

Примечание: Поиск по задачам (по строке или дате) — не реализован. Это подтверждается флагом Search = false в настройках тестов.

Архитектура базы данных

Таблица scheduler:

Поля:
- id: INTEGER — уникальный ID задачи
- date: CHAR(8) — дата в формате YYYYMMDD
- title: VARCHAR(255) — заголовок задачи
- comment: TEXT — комментарий (необязательный)
- repeat: VARCHAR(128) — правило повторения (например, daily, 5, weekly)

SQL-скрипт создания таблицы:

CREATE TABLE scheduler (
id INTEGER PRIMARY KEY AUTOINCREMENT,
date CHAR(8) NOT NULL DEFAULT '',
title VARCHAR(255) NOT NULL,
comment TEXT,
repeat VARCHAR(128)
);

CREATE INDEX IF NOT EXISTS scheduler_date ON scheduler(date);

Запуск проекта локально

Требования:
- Go 1.19 или выше
- ОС: Linux, macOS, Windows

Сборка и запуск:
go build -o scheduler cmd/scheduler/main.go
./scheduler -port=7540 -db=scheduler.db

Флаги:
-port — Порт HTTP-сервера (по умолчанию: 7540)
-db   — Путь к файлу базы данных (по умолчанию: scheduler.db)

При первом запуске автоматически создаются база данных и таблицы.

Открытие в браузере:
После запуска откройте в браузере:
http://localhost:7540

Если реализован фронтенд — будет доступен интерфейс. Иначе можно использовать API через curl, Postman или аналоги.

go test ./...

var Search = false        // Поиск по задачам не реализован

Тесты рассчитывают на то, что сервер будет доступен на порту 7540. Убедитесь, что основной сервер не запущен во время тестирования, чтобы избежать конфликта портов.

Структура проекта

Проект включает следующие ключевые директории:

- cmd/scheduler/ — точка входа (main.go)
- pkg/db/ — работа с базой данных (инициализация, запросы)
- tests/ — набор автотестов для проверки корректности API
- web/ — статические файлы фронтенда (HTML, CSS, JS)

Фронтенд:
- Директория web содержит клиентскую часть приложения.
- Сервер должен отдавать файлы из этой папки по маршруту / (корень).
- Пример: при заходе на http://localhost:7540 пользователь видит веб-интерфейс для работы с задачами.

Тесты:
- Тесты из папки tests` проверяют:
- Добавление, получение, редактирование и удаление задач.
- Корректность вычисления следующей даты при повторении.
- Работу с пагинацией (ограничение количества задач).
- Запуск тестов требует, чтобы веб-сервер был доступен на указанном в settings.go порту.
